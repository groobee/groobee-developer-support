# Groobee 스크립트 설치 가이드 - 추천 캠페인 사용법

이 문서는 웹 환경에서 Groobee 스크립트를 **실제 서비스에 맞게 설치**하기 위한 가이드 중 Groobee 추천 캠페인 사용 방법을 기술한 문서입니다.

---

## 사전 요구 사항

- (필수) 공통 스크립트가 설치되어 있어야 합니다.    
👉 [공통 스크립트 설치 가이드](../installation/installation-web-common-script.md)
- (권장) 회원 정보 연동이 되어 있어야 합니다.  
👉 [회원 정보 연동 가이드](../installation/installation-web-member-data.md)
- (권장) 행동 이력 수집 코드가 구현 되어 있어야 합니다.  
👉 [행동 이력 수집 가이드](../installation/installation-web-action.md)

> 회원 정보와 행동 이력이 수집되지 않으면, 추천 품질이 저하될 수 있습니다.

## 커스텀 웹 사이트

- [DIV형](#1-div형) \- Groobee 디자인을 약간 수정해 적용하는 방식 (임대형 쇼핑몰에 적합)
- [스크립트형](#2-스크립트형) \- 페이지 로딩 시 한 번만 갱신되는 추천에 적합
- [데이터호출형](#3-데이터호출형) \- 추천 API를 호출해 응답을 기반으로 직접 렌더링
- [데이터호출형 \- 모든페이지](#4-데이터호출형---모든-페이지-데이터호출형과-사용-방법이-다르니-주의해서-봐주세요) \- 페이지 제한 없이 키 단위로 데이터 호출이 가능한 방식

### 유형별 사용 방식

#### 1) DIV형
- **대상**: 임대형 쇼핑몰, 또는 마크업 영역에 자동 삽입이 필요한 경우
- **방식**: 추천 노출 위치에 DIV 컨테이너를 배치하고, 추천 스크립트가 자동 렌더링
- **노출 시점**: Groobee 스크립트 실행 시점에 노출되며 노출 위치에 캠페인키를 ID를 가진 DIV 컨테이너가 존재해야 합니다.
- **준비물**: 추천 노출 영역의 DOM 위치와 스타일
- **참조**: [상세 가이드](../detail/web-recommend-div.md)

#### 2) 스크립트형
- **대상**: 추천 결과를 직접 그리는 커스텀 UI
- **방식**: 페이지에 추천 스크립트를 삽입해 자동 렌더링
- **노출 시점**: Groobee 스크립트 실행 시점에 노출되며 setGroobeeRecommend 함수가 삽입되어 있어야 합니다.
- **준비물**: 스크립트 삽입 위치, 추천 영역의 레이아웃 
- **참조**: [상세 가이드](../detail/web-recommend-script.md)

#### 3) 데이터호출형
- **대상**: 추천 결과를 직접 그리는 커스텀 UI
- **방식**: 추천 API로 데이터를 호출하고, 응답 데이터를 기반으로 직접 렌더링
- **노출 시점**: 제한 없이 API를 통해 호출 가능하며 추천 캠페인에서 설정된 페이지 기반으로 호출이 가능합니다.
- **준비물**: 호출 파라미터(캠페인 키), 응답 파싱 및 렌더링 로직
- **참조**: [상세 가이드](../detail/web-recommend-data-request.md)

#### 4) 데이터호출형 - 모든 페이지 (데이터호출형과 사용 방법이 다르니 주의해서 봐주세요.)
- **대상**: 추천 결과를 직접 그리는 커스텀 UI
- **방식**: 추천 API로 데이터를 호출하고, 요청 데이터를 직접 제어, 페이지 상관없이 호출 가능, 추천 API로 데이터를 키 기반으로 호출하고, 응답 데이터를 기반으로 직접 렌더링
- **준비물**: 호출 파라미터(캠페인 키, 데이터 키, 데이터 값), 응답 파싱 및 렌더링 로직
- **참조**: [상세 가이드](../detail/web-recommend-data-request-all.md)


> 실제 파라미터/예시는 각 유형별 상세 가이드를 참고하세요.
